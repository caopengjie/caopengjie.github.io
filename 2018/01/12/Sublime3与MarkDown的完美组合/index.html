Sublime3与MarkDown的完美组合

发表于 2018-01-12 | 分类于 CodingLearning | 本文总阅读量 次

Sublime3文本编辑器

  1. Sublime3是一款比较出色的文本编辑器，到目前为止，出现过很过出色的文本编辑器。例如使用的比较多的VIM、ATOM、NOTEPAD++、TextPad等。它们都有各自的特色。对于Sublime而言，它具有各种功能，是一个顶尖的代码、标记语言、文本编辑器。从而成为了广大程序员的热爱。

  2. Sublime3的获取地址：http://www.sublimetext.com/, 直接下载安装既可以使用。Sublime是一个收费软件，但是可以下载使用。不需要注册破解，但是每过一段时间都会提示你购买正版，会有不好的体验。这里我提供一个比较新的注册码，亲测可以使用。破解流程，复制注册码到Help->LICENSE既可以免费使用。小编这里建议大家还是多多支持正版。

    Sublime Text3 3143注册码：

  3. Sublime相关主题、插件的安装。这里介绍几款必须安装的插件，Package Control，这个插件是用来管理相关插件的。

于此同时还有相关的一些主题插件，以及一些高亮代码插件。例如Material-Theme、Boxy、SideBar等。这些插件在Package Control:install Package中都能够安装，比较方便。

MarkDown标记语言

本人因为在接触写博客的时候，急需一种简单的简单的文本标记语言来排版构写。因此MarkDown就成为了我的首选。Markdown是一种轻量级标记语言，它让作者更多的注重文章的内容而不是文章的样式。且其比较简单，很容易入手。具体的语法说明可以访问其中文语法网站:http://wowubuntu.com/markdown/。小编这里主要是列举一些MarkDown没有一些实现的语法处理以及常用的一些操作。

实现段首空格的显示:

半方大的空白`&ensp;或&#8194`;<br>
全方大的空白`&emsp;或&#8195`;<br>
不断行的空白`格&nbsp;或&#160`;

实现文字换行：

直接回车不能换行，可以在上一行文本后面补两个空格或者是在两行文本直接加一个空行。
再者因为MarkDown兼容HTML的缘故，因此可以直接使用标记<br>，从而达到换行的目的。

实现图片插入：

对于在MarkDown插入图片，它有固定的语法`![介绍][图片链接]`，但是对于一些技术博主构写技术博客的时候插入相关的截图说明就没有那么
方便。这里有大多数人会选择首先将截图上传到相关的图床平台，获取相应的图片链接，然后填入进去。这里我介绍一下将截图发布到github上，
生成相应的图片链接。
1. 在github上新建一个项目，建立一个新的仓库MarkdownPhotos。
2. 在本地电脑新建一个存放图片的仓库，并将自己的一些截图放入到里面。
3. 然后将本地的图片push到github上。

实现目录：
对于Markdown文本语言标记使用[TOC]来实现，但是本人尝试了许多次后并没有实现。在这里希望实现成功的朋友可以和我多多交流一下。
对于目录生成，我用MarkdownTOC来实现，在下一段将会有提到。

MarkDown在sublime上的配置

这里本人推荐一款强大的插件OmniMarkupPreviewer，它可以支持Markdown标记语言在Sublime上的编写，同时还支持将标记语言渲染为HTML并在浏览器上实时预览(这个功能超级赞)，这就为博文撰写者提供了极大的便利。

  1. 安装：在Package Control直接Install Package,并搜索该差价直接安装就可以。

  2. 相关设置，相关的设置信息可以到Preferences->Package Settings->OmniMarkupPreviewer->Settings-Default中查看，其默认的设置已经能满足大部分需求。

  3. 实时浏览器预览功能，这个功能比较实用。快捷键CTR+ALT+O即可以预览，这里装完插件可以能会遇到无法预览的问题。

    Sorry, the requested URL ‘http://127.0.0.1:4550/view/52‘ caused an error:
    ‘buffer_id(52) is not valid (closed or unsupported file format)’
    NOTE: If you run multiple instances of Sublime Text, you may want to adjust
    the server_port option in order to get this plugin work again.

解决的办法如下，在Preferences->Package Settings->OmniMarkupPreviewer->Settings-User添加如下信息：

{
    "renderer_options-MarkdownRenderer": {
    "extensions": ["tables", "fenced_code", "codehilite"]
  }
}

  1. Markdown目录插件MarkdownTOC，这个在Install Package中直接搜索就可以安装。本人是在尝试[TOC]标签无用的情况下，选择使用这款插件来实现目录功能。下载后，需要在在Preferences->Package Settings->OmniMarkupPreviewer->Settings-User设置一些信息：

    {
    “default_autolink”: true,
    “default_bracket”: “round”,
    “default_depth”: 0
    }

生成目录，只需要选中相应的标题，在Tools->MarkdownTOC->Insert TOC,更新目录则选择Tools->MarkdownTOC->Update TOC。
